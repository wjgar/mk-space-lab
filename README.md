# mk-space-lab

## 项目简介

`mk-space-lab` 是一个用于归档和发布多个独立 HTML 页面的仓库。  
这里存放的页面内容广泛，不针对任何单一域名或具体网站，主要用于实验性页面展示、静态内容存档以及轻量级前端示例整理。

## 目录说明

```
mk-space-lab/
├── index.html          # 仓库主页 / 导航入口
├── pages/              # 独立 HTML 页面存放目录
│   ├── example-1.html
│   ├── example-2.html
│   └── ...
├── assets/             # 公共资源（样式、脚本、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `index.html`：仓库的默认入口页面，提供页面列表或导航。
- `pages/`：所有独立 HTML 页面均放置于此，每个文件为一个独立的页面归档。
- `assets/`：存放页面共用的样式表、脚本和图片资源，便于统一管理。

## 页面归档说明

本仓库的页面遵循以下归档原则：

- 每个 HTML 文件均为独立的、可自包含的页面（除非引用 `assets/` 下的公共资源）。
- 页面内容可能包含实验性功能、静态展示、示例代码或信息归档，不代表任何正式网站或服务。
- 页面之间无强关联，各自独立维护。
- 所有页面均为静态内容，不依赖服务器端动态处理。

## 维护说明

- 新增页面：在 `pages/` 下创建新的 `.html` 文件，并在 `index.html` 中添加链接。
- 资源更新：如需修改公共样式或脚本，请更新 `assets/` 下对应文件，并确保不影响已有页面。
- 删除页面：直接从 `pages/` 移除文件，并更新 `index.html` 中的导航。
- 本仓库不涉及用户数据收集、跟踪代码或第三方服务调用，仅用于页面展示与归档。

## 其他

- 本仓库为个人或团队内部使用，也欢迎根据需要进行 fork 或参考。
- 如发现页面内容存在错误或链接失效，欢迎提交 Issue 或 Pull Request。
